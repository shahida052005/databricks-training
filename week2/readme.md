# Week 2 SQL Practice

This week I expanded my SQL practice into more advanced and structured topics. The work covered CMS-style queries, joins, NULL handling functions, regular expressions, and string to number extraction using real practice datasets.

---

## Week 2 Summary

- Day 1: CMS queries and practice using the CMS starter schema.
- Day 2: Join based SQL practice.
- Day 3: NULL function practice using ISNULL, IFNULL, COALESCE, and NULLIF.
- Day 4: Regular expression queries for extracting text, numbers, email parts, and phone patterns.
- Day 5: String and numeric data extraction practice.

---

## Day 1 Work

On day 1, I worked on CMS queries using a starter database. This helped me practice filtering, selecting, and working with structured records in a more application-like schema. It strengthened my confidence in writing queries against a realistic dataset and understanding how table relationships are used in common project scenarios.

---

## Day 2 Work

On day 2, I practiced SQL joins using a dedicated schema file. The exercises helped me work with related tables and understand how to combine data from multiple tables using join operations. This was useful for identifying matching and unmatched records and for building stronger foundation in relational querying.

---

## Day 3 Work

On day 3, I focused on NULL functions and null-safe SQL logic. I practiced handling missing values using functions such as IFNULL, COALESCE, and NULLIF, along with filtering rows containing NULL values. This improved my understanding of how SQL behaves when data is incomplete and how to write safe queries for real world datasets.

---

## Day 4 Work

On day 4, I worked on regular expressions in SQL. I wrote queries to extract numeric values, text fragments, domains from email addresses, and patterns from phone numbers and mixed strings. This helped me understand how regex can be used for advanced pattern matching and text processing inside SQL queries.

---

## Day 5 Work

On day 5, I practiced string and numeric data extraction using combined text patterns. The queries focused on pulling specific parts from mixed values, names, phone numbers, and structured text fields. This gave me more confidence in handling formatted strings and extracting useful information from unstructured or semi-structured data.

---

## Skills Covered

- Basic SELECT and WHERE queries.
- SQL joins and relational mapping.
- NULL handling with IFNULL, COALESCE, and NULLIF.
- Regular expressions in SQL.
- String parsing and extraction.
- Working with structured and semi structured data.

---

## Outcome

By the end of Week 2, I had improved my ability to handle real world SQL problems involving joins, missing values, regex based extraction, and string manipulation. This week helped me move beyond basic querying and build stronger problem solving skills for production style SQL tasks.
